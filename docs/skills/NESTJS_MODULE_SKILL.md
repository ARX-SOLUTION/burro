# Nestjs Module Skill

## Scope
Reusable skill instructions for nestjs_module_skill tasks.

## Inputs
- Product docs in `../` or `../../` as applicable.
- Existing source files in apps and packages.
- User requests and acceptance criteria.

## Outputs
- Updated implementation or documentation.
- Notes on tests and risks.

## Rules
- Use pnpm only.
- Inspect existing files before changes.
- Keep documentation links relative.
- Update API docs when endpoint contracts change.

## Do Not
- Do not commit secrets.
- Do not delete existing work without explicit confirmation.
- Do not invent production behavior beyond documented scope.

## Test Checklist
- Relevant typecheck passes.
- Relevant build passes or exact blocker is documented.
- Docs links remain relative.
